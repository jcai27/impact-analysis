Engineering Impact Report
Methodology Impact is quantified across five dimensions: feature delivery (35%) — commits classified as features, weighted by complexity · system centrality (25%) — frequency with which an engineer touches files that are broadly shared across the codebase · code ownership (20%) — fraction of commits to modules where the engineer is the primary contributor · quality maintenance (15%) — bug fixes, refactors, and test coverage · collaboration (5%) — unique co-contributors on shared files. All scores are normalised relative to the full engineer cohort in the selected window.
